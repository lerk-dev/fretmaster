## 1. 产品概述对比

| 维度       | SOLO Guitar Trainer                          | FretMaster              |
| -------- | -------------------------------------------- | ----------------------- |
| **定位**   | 专业吉他指板视觉化练习工具                                | 基于 SOLO 理念的开源复刻/致敬版本    |
| **开发团队** | Trio Software Ltd（Tom Quayle & David Beebee） | 开源社区（lerk-dev）          |
| **价格**   | 付费应用（$9.99 / €17.99）                         | 完全免费开源                  |
| **平台**   | iOS, iPadOS, macOS, Android                  | Web, Windows（EXE）       |
| **技术栈**  | 原生移动应用                                       | Next.js + React + Tauri |
| **设计理念** | 专注和声方面，排除节奏元素                                | 复刻 SOLO 核心功能，增加调音器等工具   |

### 核心设计理念差异

**SOLO**:

- 专注于「和声」层面的指板视觉化
- 排除节奏和时间元素，让用户按自己的节奏练习
- 强调"授人以渔"，不直接提供大量指法图
- 使用「音程功能数字」（Intervallic Functions）而非具体音名

**FretMaster**:

- 复刻 SOLO 的核心练习模式
- 新增「调音器」实用工具
- 保留音名显示选项（与 SOLO 的纯数字理念不同）
- 增加 Web 端的便捷性和可部署性

## 4. 数据处理流程对比

### 4.1 音频处理流程

**SOLO（原生应用）**：

```
麦克风输入 → 原生音频 API → 音高检测算法 → 结果匹配 → 视觉反馈
```

**FretMaster Web 版**：

```
麦克风输入 → Web Audio API → ScriptProcessorNode → YIN 算法 → 结果匹配 → 视觉反馈
```

**FretMaster EXE 版**：

```
麦克风输入 → Tauri 原生音频 → Rust 音频处理 → YIN 算法 → 结果匹配 → 视觉反馈
```

### 4.2 数据存储对比

| 数据类型      | SOLO          | FretMaster Web           | FretMaster EXE |
| --------- | ------------- | ------------------------ | -------------- |
| **练习记录**  | 本地 SQLite     | localStorage / IndexedDB | SQLite 数据库     |
| **用户设置**  | 应用内存储         | localStorage             | SQLite + 配置文件  |
| **自定义进行** | 应用内存储         | localStorage             | SQLite         |
| **统计数据**  | 基础统计          | 详细统计 + 图表                | 详细统计 + 图表      |
| **数据导出**  | 有限            | 支持 JSON 导出               | 支持 JSON 导出     |
| **数据备份**  | iCloud/Google | 手动导出                     | 文件备份           |

### 4.3 练习数据流

**SOLO**：

```
用户输入 → 音频检测 → 音高识别 → 音程功能计算 → 正确性判断 → 记录统计
```

**FretMaster**：

```
用户输入 → 音频检测 → 音高识别 → 音符映射 → 音程功能计算 → 正确性判断 → 记录统计 → 更新图表
```

## 7. 用户操作流程对比

```

```

```

```

```
打开应用/网页 → 选择练习模式 → 配置选项 → 开始 → 练习 → 查看统计
```

| 维度       | SOLO  | FretMaster |
| -------- | ----- | ---------- |
| **专业性**  | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐       |

| **功能性**  | ⭐⭐⭐⭐  | ⭐⭐⭐⭐⭐      |
| **性能**   | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐       |
| **可访问性** | ⭐⭐⭐   | ⭐⭐⭐⭐⭐      |
| **性价比**  | ⭐⭐⭐   | ⭐⭐⭐⭐⭐      |
| **可定制性** | ⭐⭐    | ⭐⭐⭐⭐⭐      |

### 9.2 各版本适用场景

| 场景             | 推荐选择                  |
| -------------- | --------------------- |
| **专业吉他手日常练习**  | SOLO（iOS/Android）     |
| **初学者入门学习**    | FretMaster Web（免费）    |
| **Windows 用户** | FretMaster EXE        |
| **路由器本地部署**    | FretMaster Web        |
| **自定义需求**      | FretMaster（开源）        |
| **无网络环境**      | SOLO 或 FretMaster EXE |

### 9.3 FretMaster 改进建议

#### 高优先级

1. **优化音频延迟**：Web 版音频延迟较高，建议使用 EXE 版本进行严肃练习
2. **增加 Two-Point System 教学**：这是 SOLO 的核心理念，建议增加相关教学内容
3. **完善 iOS/Android 支持**：考虑开发 PWA 或原生应用

#### 中优先级

1. **增加触觉反馈**：移动端支持震动反馈
2. **优化离线体验**：Web 版完善 Service Worker 缓存
3. **增加练习课程**：结构化练习计划，类似 SOLO 的课程体系

#### 低优先级

1. **社交功能**：练习成绩分享、排行榜
2. **更多乐器支持**：尤克里里、曼陀林等
3. **AI 练习建议**：基于练习数据的智能推荐

### 9.4 SOLO 优势保持点

1. **原生音频低延迟**：移动端的音频性能优势
2. **专业教学设计**：Two-Point System 等核心教学方法
3. **品牌课程集成**：与 Ultimate Fretboard Visualization 课程深度集成
4. **跨平台同步**：iCloud/Google 数据同步
5. **成熟稳定**：经过多年迭代，稳定性高

### 9.5 FretMaster 独特价值

1. **完全免费开源**：零成本使用，可自由修改
2. **Web 可部署**：可在任何设备通过浏览器访问
3. **中文支持**：完整的中文界面和文档
